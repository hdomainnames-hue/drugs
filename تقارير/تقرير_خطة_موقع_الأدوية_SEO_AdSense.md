# تقرير خطة إنشاء موقع الأدوية (SEO + AdSense) — عربي/إنجليزي — Next.js + Vercel + Neon

## 1) الهدف
إنشاء موقع أدوية مستقل بالكامل (لا علاقة له بمشروع المستشفيات) يحقق:
- صفحات قابلة للزحف والفهرسة بشكل ممتاز.
- جاهزية عالية للموافقة في Google AdSense.
- تجربة مستخدم حديثة وسريعة على الموبايل والديسكتوب.
- دعم ثنائي اللغة (عربي/إنجليزي) مع دعم RTL كامل للعربية.
- قابلية نمو لاحقًا (بحث أقوى، محتوى أكثر، تحسينات تقنية).

## 2) أهم قرار تقني لتجنب خطأ الموقع السابق (SPA)
الخطأ السابق: الموقع كان ديناميكي/SPA ويعتمد على JavaScript في المتصفح، مما يؤدي إلى:
- صعوبة الزحف والفهرسة.
- ضعف توافق AdSense.

القرار الصحيح هنا:
- صفحات الدواء/الشركات/المواد/المقالات يجب أن تكون **HTML جاهز على السيرفر** (SSG/ISR أو SSR) بحيث يظهر المحتوى الأساسي بدون أي JavaScript.
- صفحات البحث يمكن أن تكون تفاعلية، لكن تُعامل كصفحات غير مفضلة للفهرسة.

## 3) الاستضافة والميزانية (مجاني حاليًا)
- **Vercel**: نشر مجاني كبداية.
- **Neon Postgres**: قاعدة بيانات Postgres مجانية كبداية.

ملاحظة: "مجاني" يعني علينا تجنب حلول تتطلب سيرفر دائم أو محرك بحث خارجي مدفوع.

## 4) التقنية المقترحة (Stack)
- **Next.js (App Router)**
  - يوفر SSG/ISR/SSR.
  - `metadata` و`generateStaticParams` و`robots` و`sitemap` بشكل رسمي.
- **TypeScript** (لمنع الأخطاء وتسهيل الصيانة).
- **TailwindCSS** للتصميم الحديث سريع التطوير.
- **نظام ثيم**: Light/Dark.
  - Light ليس أبيض فاقع، بل Off-white مع تباين جيد.
- **ORM**: Prisma أو Drizzle (يفضل Prisma كبداية مع Neon).

## 5) i18n (عربي/إنجليزي) — متطلبات دقيقة
### 5.1 هيكلة الروابط
- العربية: `/ar/...`
- الإنجليزية: `/en/...`

### 5.2 RTL/LTR
- العربية: `dir="rtl"` + خطوط مناسبة + ضبط spacing.
- الإنجليزية: `dir="ltr"`.

### 5.3 ترجمة الواجهة
- كل نصوص الواجهة (القوائم، الأزرار، العناوين العامة، رسائل الأخطاء) تُترجم بالكامل.

### 5.4 ترجمة محتوى الدواء
- لا يُنصح بترجمة أسماء الأدوية التجارية آليًا (قد تسبب أخطاء).
- الأفضل:
  - الاسم كما هو.
  - إضافة محتوى عربي أصلي (FAQ/شرح) يدعم السيو.
  - إضافة محتوى إنجليزي أصلي مختصر لاحقًا.

## 6) بنية الصفحات (IA) وSEO URLs
### 6.1 صفحات تُفهرس (Indexable)
- **تفاصيل دواء**
  - `/ar/drug/[slug]-[id]`
  - `/en/drug/[slug]-[id]`
- **شركة**
  - `/ar/company/[slug]`
  - `/en/company/[slug]`
- **مادة فعالة**
  - `/ar/active-ingredient/[slug]`
  - `/en/active-ingredient/[slug]`
- **مقالات/أدلة**
  - `/ar/blog/[slug]`
  - `/en/blog/[slug]`
- **صفحات الثقة والسياسات**
  - `/ar/privacy` `/ar/terms` `/ar/about` `/ar/contact` `/ar/disclaimer`
  - ونفسها للإنجليزية.

### 6.2 صفحات غير مفضلة للفهرسة
- `/ar/search?q=...` و`/en/search?q=...`
  - إعداد `noindex, follow` عادةً.

## 7) قاعدة البيانات (Neon Postgres) — نموذج بيانات مقترح
### 7.1 جداول أساسية
- `drugs`
  - `id` (PK)
  - `remote_id` (UNIQUE)
  - `slug_ar`, `slug_en`
  - `name` (اسم تجاري)
  - `company`
  - `active_ingredient`
  - `price`
  - `description_raw` (من السحب)
  - `description_ar` (محسن للسيو)
  - `description_en` (اختياري)
  - `meta_description_ar`, `meta_description_en`
  - `image_path` (مسار الصورة)
  - `updated_at`

- `drug_similar`
  - `drug_remote_id`
  - `similar_remote_id`
  - UNIQUE(drug_remote_id, similar_remote_id)

- `articles`
  - `id`, `lang`, `slug`, `title`, `excerpt`, `content_md`, `published_at`

- `faqs`
  - `id`, `lang`, `drug_remote_id` (nullable), `article_id` (nullable), `question`, `answer`

### 7.2 البحث
- الآن: بحث Postgres LIKE + Index.
- لاحقًا: Full-text (GIN) أو Meilisearch/Typesense عند توفر ميزانية.

## 8) الصور
- الأفضل للأداء: WebP (أو AVIF) + أبعاد ثابتة + Lazy-loading.
- لمرحلة مجانية:
  - نبدأ باستراتيجية بسيطة ثم نقرر أفضل تخزين (Vercel/Storage) حسب حجم الصور.

## 9) التصميم والواجهة (UX)
### 9.1 المتطلبات
- تصميم حديث متجاوب.
- Light/Dark.
- ألوان غامقة ومتباينة حتى في Light.
- Bottom Navigation في الموبايل.

### 9.2 صفحات أساسية
- الرئيسية: بحث + أقسام (شركات/مواد/مقالات) + روابط مهمّة.
- صفحة دواء:
  - صورة + بيانات أساسية + أقسام (الوصف، بدائل/مشابهات، FAQ، تحذير طبي).
- صفحات قائمة شركات/مواد:
  - صفحات فهرسة قوية + روابط داخلية.

## 10) SEO Checklist (تقني)
- Canonical لكل صفحة.
- `hreflang` للـ AR/EN.
- OpenGraph/Twitter cards.
- JSON-LD:
  - BreadcrumbList
  - FAQPage عند توفر FAQ
- `sitemap.xml` index + تقسيم.
- `robots.txt` مضبوط.
- 404 صفحة جيدة.
- Core Web Vitals:
  - تقليل JS
  - ضغط الصور
  - caching

## 11) AdSense Checklist (القبول)
- صفحات السياسات مكتملة (AR/EN).
- محتوى أصلي (مقالات/FAQ) وليس فقط نسخ.
- تجربة ممتازة على الموبايل.
- تقليل الإعلانات في البداية.
- تواصل واضح + صفحة من نحن.

## 12) مراحل التنفيذ المقترحة (High-level)
1) تأسيس المشروع + i18n + الثيم + Bottom bar.
2) إعداد Neon + ORM + استيراد البيانات.
3) بناء صفحات SEO الأساسية (دواء/شركة/مادة/مقالات).
4) إنشاء sitemaps + robots + صفحات السياسات.
5) تحسين الأداء + إضافة محتوى أصلي + جاهزية AdSense.

## 13) ملاحظات ختامية
- هذا المشروع مستقل بالكامل عن أي مشروع آخر.
- أي قرار في الواجهة أو الداتا يتم تقييمه وفق هدف: SEO + AdSense + سرعة + تجربة المستخدم.
